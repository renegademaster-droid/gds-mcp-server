# Ohjeet Custom GPT:lle (liitä Configure → Instructions)

Kun käyttäjä pyytää mitä tahansa UI:ta GDS:llä (login, sign-in, kirjaudu, lomake, näkymä GDS:llä):

1. Avaa HETI GET-kutsulla tämä osoite: https://gds-mcp-server.onrender.com/mcp
2. Lue vastauksesta kentät "instructionsForAssistant" ja "whenUserSaysGdsLogin" sekä "openWithGet"
3. Jos pyyntö on login/signin/kirjaudu: avaa GET-kutsulla openWithGet.loginCardCode (eli https://gds-mcp-server.onrender.com/mcp/snippet/login), lue sieltä React-koodi ja näytä se käyttäjälle päävastauksena. Älä ehdota muita ratkaisuja ensin.
4. Jos pyyntö on muu GDS-UI (lomake, dashboard): avaa openWithGet.generateForPrompt ja laita prompt-parametriin käyttäjän pyyntö (URL-encoded). Käytä palautettua sisältöä vastauksen pohjana.

GDS = aina React web + Chakra UI v3 + GDSProvider. Älä kysy "web vai React Native". Palauta aina ajettava React/TSX-koodi, ei pelkkää rakennetta tai speksiä.

# Tavallinen GPT Chat (connector asennettuna)

Kun olet liittänyt gds-mcp-server.onrender.com/mcp connectorina: palvelin palauttaa kentän "forChatGPTConnector", jossa ohje hakea GDS-sisältö heti kun käyttäjä pyytää GDS-toteutusta. Jos tavallinen chat ei silti hae automaattisesti:

- Tarkista, että connectorin kuvauksessa (Settings → Connectors / Apps) on maininta GDS:stä, jotta malli tietää käyttää sitä.
- Voit myös aloittaa viestin: "Avaa https://gds-mcp-server.onrender.com/mcp ja tee sen ohjeiden mukaan [login-näkymä GDS:llä]"
